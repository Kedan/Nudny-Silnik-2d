glm::vec2 Layer::GetCorrectTilePosition ( tson::Vector2f t_obj_position,
glm::vec2 t_tile_size )

IF you use tiles with size ( i.e. 64x80 pixels ) different than map grid ( i.e. 16x16 pixels ), tson::Object.getPosition() gives you shit becouse it multiplies tile size, not the map grid size, so when tile is position ( 1, 1 tiles) - it gives (1*64, 1*80 pixels ) instead of (1*16, 1*16)

Parameters
t_obj_position[tson::Vector2f]. Tileson object position - tson::Object::getPosition() in pixels
t_tile_size[glm::vec2] actual tile size in pixels
Returns
(glm::vec2) Corrected tile position, calculates using actual map grid size.
